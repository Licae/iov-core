# 需求管理 / TARA / 测试用例联动待办

## P0（本周必须完成）

- 建立追溯基线数据
  - 每个核心安全域至少有 1 条需求（访问控制、身份认证、口令策略、日志安全、网络暴露、OTA 安全）。
  - 每条需求至少关联 1 条 TARA 与 1 条可执行测试用例。
- 固化联动流程
  - 新增测试用例时，必须选择关联需求与 TARA（不允许“裸用例”进入基线套件）。
  - 删除需求/TARA 时给出影响提示（受影响用例数）。
- 执行入口联动验证
  - 发起任务时可按“需求”或“TARA”筛选可执行用例。
  - 任务详情增加“来源需求/威胁”展示，支持一键回溯。
- 结果口径统一
  - 执行状态：`PENDING/RUNNING/COMPLETED/CANCELLED`
  - 测试结果：`PASSED/FAILED/BLOCKED/ERROR`
  - UI 与导出报告统一同一套枚举与颜色。

## P1（下一阶段）

- 基线套件版本化
  - 建立“需求基线版本”和“TARA 基线版本”，支持差异对比与回滚。
- 证据汇聚
  - 用例执行完成后自动沉淀“命令、退出码、stdout/stderr、结论”并挂接到需求/TARA 节点。
- 风险热力图
  - 按 TARA 风险等级统计覆盖率：`已覆盖用例数 / 应覆盖用例数`。

## P2（生产化）

- 变更审批流
  - 需求、TARA、用例变更需审批与审计记录。
- 自动化回归门禁
  - 针对 P0/P1 风险需求，发布前强制执行关联用例并校验通过率阈值。

## 当前建议的验收标准

- 至少 10 条核心安全用例完成“需求 + TARA”双关联。
- 关联后的用例在“用例管理”可见 `需求关联数/TARA关联数 > 0`。
- 任一需求可反查命中的 TARA 与测试用例。
- 任一 TARA 可反查命中的需求与测试用例。
